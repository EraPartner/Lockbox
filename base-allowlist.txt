# BASE egress allowlist — the hosts EVERY agent devcontainer needs.
# One host per line; a leading dot matches the domain and all subdomains.
# Lines starting with '#' and blank lines are ignored by squid.
#
# This is the shared floor. Per-project additions live in each project's
# .devcontainer/allowlist.extra.txt; `sync.sh` concatenates base + extra into
# the baked .devcontainer/allowlist.txt. Keep this MINIMAL — anything added here
# widens egress for all six containers (Brain, Vision, Watchman, Napoleon-relay,
# git-agent, sandbox). The generic sandbox keeps its dev floor (npm/PyPI) in its OWN
# allowlist.extra.txt, NOT here, so the others stay tight.

# Anthropic / Claude Code — the LLM API + auth/token refresh.
api.anthropic.com
console.anthropic.com

# GitHub — the MINIMAL floor every container needs: git over HTTPS (github.com +
# codeload.github.com for push/fetch) and the REST/GraphQL API (api.github.com for
# gh). Deliberately NOT the raw/release-asset content hosts: raw.githubusercontent
# .com and objects.githubusercontent.com are multi-tenant (ANY public repo / release
# asset), so on the shared floor a compromised in-container agent could pull
# attacker-controlled content through the egress lock. They are NOT required by git
# push/fetch — only by ad-hoc `curl` / `gh release download` — so a project that
# needs them adds them to its OWN allowlist.extra.txt (the generic `sandbox` does).
# Likewise NOT a bare `.githubusercontent.com` wildcard (that apex matches gist./
# camo./avatars./…): scope to exact hosts, per project.
github.com
api.github.com
codeload.github.com
